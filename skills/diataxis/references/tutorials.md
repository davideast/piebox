# Tutorials (Learning-Oriented)

A tutorial is a lesson that takes the beginner by the hand. It is responsible for the user's _meaningful_ acquisition of basic competence.

## Key Principles & The "Tutorial vs. How-To" Tension

- **The user is a beginner:** Assume they know nothing about this specific tool.
- **The goal is educational, not practical:** The user is here to learn, not to get a job done. The end result (e.g., "Hello World") doesn't matter; the _experience_ of building it does.
- **Control the environment:** Do not let the user make choices. Do not say "You can use PostgreSQL or MySQL." Say: "We will use SQLite."
- **Focus on success:** The tutorial must work flawlessly. The psychological goal is to build the user's confidence.
- **Inspire action:** Give the user something to do immediately.

## Writing Rules

- **Tone:** Friendly, guiding, reassuring.
- **Structure:** Chronological steps. Tell them what they will do, tell them to do it, tell them what happened.
- **Anti-patterns:** Do not explain architecture here. Do not provide reference tables. Do not offer alternatives.
